# HTML 基本文法

**HTMLの基本構文**
```
<!DOCTYPE html>
<html>
  <head>
    <title>Webページのタイトル</title>
    # Webページの情報を記載します
  </head>
  <body>
    # ブラウザに表示するコンテンツを記載します
  </body>
</html>
```

**SEOとは**
SEO（Search Engine Optimization）検索エンジン最適化の略。Googleなどの検索エンジンでウェブサイトが検索結果の上位に表示されるように、サイトの構造やコンテンツを調整する活動全般を指す。

**タグの基本文法**
`<a>タグはWebページ内でリンクを設定する際に使用する。</a>`
【基本構文】
`<a href="実際のリンク">リンクにしたいテキスト</a>`

【画像にリンクを設定する際の書き方】
`<a href="実際のリンク"><img src="画像のパス" alt="代替えテキスト"></a>`

【リンクを別タブで開く方法】
`<a href="実際のリンク" target="_blank" rel="noopener noreferrer">別タブで開くリンク</a>`
ページ遷移先として外部のサイトを設定する際には少し注意が必要で、
rel="noopener noreferrer設定がない場合では、別タブで開いた外部サイト上に万が一悪意のあるプログラムが作成されている場合に、元々のサイト（リンクを作成したサイト）がそのプログラムに操作されてしまう場合がある。
この脆弱性を防ぐために、** 別タブでページを開く際には、rel="noopener noreferrer設定を忘れないようにする。 **

**`<img>`タグにはalt属性を設定する。**
alt属性を使用すると、パスの誤りなどで画像が表示されない場合に、代わりにその画像の説明を表示することが可能になる。
alt属性は検索エンジンへの補足説明としても有効。SEO観点でも設定は必須。

**`<form>`/`<input>`タグはユーザーにテキスト入力や要素を選択してもらう場合に使用するタグ。**
`<form>`タグで入力フォームとする箇所全体を囲い、その中に`<input>`タグを配置するだけで入力フォームが作成される。`<input>`タグは終了タグは不要。
`<input>`タグのtype属性
text：テキストタイプの入力欄が表示される。
password：パスワードの入力欄が表示される。
file：ファイルをアップロードするためのボタンが表示される。
radio：ラジオボタンが表示される。
checkbox：チェックボックスが表示される。
reset：同じformタグ内の入力内容をクリアするためのりせっとボタンが表示される。
submit：同じformタグ内の入力内容を送信するためのボタンが表示される。

`<input>`タグの便利な属性
1.name属性：フォームを通じて送信されるデータを識別するためのキーとして機能する。
例）
`<form action="送信先のURL">`
      `<input type="text" name="username">`
      `<input type="password" name="password">`
`</form>`

2.placeholder属性：入力フォームの記入例を表示する。
例）
`<form action="送信先のURL">`
      `<input type="text" placeholder="記入例を書きます">`
`</form>`

3.accept属性：アップロードできるファイルの種類を指定することができる。
例）
`<form action="送信先のURL">`
      `<input type="file" accept="image/png, image/jpeg">`
`</form>`

***`<label>`タグは設定した`<input>`タグにラベルを設定するために使用され、入力フォームをより見やすく整理する。***
必須機能ではないが、一般的な入力フォームには設定が推奨される。

`<label>`タグの書き方1
`<form action="送信先のURL">`
      `<label>`
        メールアドレス`<input type="email">`
      `</label>`
`</form>`

`<label>`タグの書き方2
`<label>`タグのfor属性を使って任意の`<input>`タグを紐づける方法。
for属性と対応する値を`<input>`タグのid属性に設定する必要がある。
2つの方法で優劣は特にないが、一般的にこちらの方法が採用されることが多い。
`<form action="送信先のURL">`
      `<label for="email">メールアドレス</label>`
      `<input type="email" id="email">`
`</form>`

***`<label>`タグを使用するメリット***
`<label>`タグを使用すると、`<label>`タグをクリックするだけで、そこに紐づいた入力フォームにフォーカスを当てることができる。
また、チェックボックスやラジオボタンの場合、ボタンが小さくなりがちだが、その場合も`<label>`タグをクリックすることで任意の選択肢を選択をすることができる。

**`<textarea>`タグとは**
このタグは、ユーザーからのフィードバック、コメント、その他のテキストベースの入力を受け取る際によく使用される。

`<textarea name="comment"></textarea>`

`<textarea>`タグの書き方その３
rowsとcols属性を使用して、`<textarea>`のサイズを指定する。

**`<button>`タグ**
`<button`タグはButton(ボタン)を意味するタグであり、Webページにボタンを表示したいときに使用する。
Webサイト上では、フォームの送信ボタンやリセットボタンを設置するために使われることが多く、一般的なWebアプリケーションでは必ずと言ってよいほど登場する重要なタグ。
【基本構文】
`<button type="ボタンの種類">ボタンの文言</button>`

**type属性の設定値**
submit：送信用のボタンが作成される。フォームに入力された値が送信される。
reset：リセットボタンが作成される。フォームに入力された値が取り消される。
button：通常のボタンが作成され、クリックしても特に何も発生しない。onclick属性にJavaScriptのイベントを設定し、クリック後の処理はそっちで実装していく。

**`<button>`タグのその他の属性**
1. disabled属性
   disabled属性を設定すると、ユーザがそのボタンをクリックできなくなる。入力必須の項目が入力されていない場合に設定することが多い。
   使用例)`<button type="submit" disabled>クリック不可</button>`

2. name/value属性
   name属性にキー、value属性に値を設定することで設定値を送信することが可能になる。

**`<button>`タグと`<input>`タグの違い**
1. buttonタグは子要素を持てる。inputタグは終了タグがない。そのため、buttonタグの内側に他のタグを設定することも可能。
2. buttonタグは疑似要素を持てる。

**`<ul>`/`<li>`タグ**
ul、liタグはそれぞれ、「Unordered List(順不同のリスト)」, 「List Item(リストアイテム)」を略したタグであり、任意の項目をリスト形式で表示したいときに使用する。例えば、Webサイトのナビゲーションなど、サイト上のコンテンツを一覧で表示する際に活躍する大事なタグ。
`<ul>`タグと`<li>`タグは必ずセットで使用される。
【基本構文】
`<ul>`
      `<li>項目その１</li>`
      `<li>項目その２</li>`
      `<li>好きな数だけ追加できます。</li>`
`</ul>`

ulタグを使用すると、デフォルトで「・」のマークがつき、改行後のインデントを自動で揃えてくれる。また、CSSを使用することで、リストを横並びにしたり、「・」を変更したり非表示にすることも可能。
なお、横並びにするときは、`ul { display : flex; }`でできる。
flex: Flexboxの略。

「・」を非表示にするには、`ul { list-style: none;}`でできる。

**順序付きリストを作成するなら`<ol>`と`<li>`タグを使用**
```
<ol>
  <li>項目その1</li>
  <li>項目その2</li>
  <li>項目その3</li>
</ol>
```

**表示結果**
```
1.項目その1
2.項目その2
3.項目その3
```

```
divタグ：「division」の略、HTML文書内でのセクションやコンテンツを区分けするために使用される。
divタグはスタイルやレイアウトを適用するためのコンテナとしてよく使われるが、
それ自体には意味的な内容はない。主にCSSやJavaScriptと組み合わせて、
ページの特定の部分にスタイルや動作を適用する際に利用される。
```
```
sectionタグ：セマンティック（意味的）な要素で、
文書やアプリケーションの中の独立したセクションを定義するために使用される。
例えば、章、ヘッダー、フッター、または文書の他のセクションなど。
セクションタグは関連するコンテンツをまとめるために使用され、
それぞれのセクションが独自の意味を持つべきである。
```
```
articleタグ：これもセマンティックな要素で、文書内の独立した、自己完結型のコンテンツを表す。

```

これらのタグは、ウェブページの構造を明確にし、検索エンジンやスクリーンリーダーなどのユーザーエージェントに対してコンテンツの意味をより適切に伝えるために使われる。また、セマンティックなHTMLはアクセシビリティを改善し、検索エンジン最適化（SEO）にも役立つ。

**`<div>`タグの基本的な使い方**
`<div>` タグは、HTMLドキュメント内でコンテンツのブロックを作成し、スタイルやレイアウトの適用に広く使われる非常に汎用的なコンテナ。
```
<div>
   <!-- ここにコンテンツを配置 -->
</div>
```

**`<section>`タグの基本的な使い方**
`<section>` タグは、HTML5で導入されたセマンティック（意味的な）要素で、関連するコンテンツをまとめるためのセクションを定義する際に使用されます。これは、ウェブページ内の異なる部分が独自の意味を持つべき場合に特に役立つ。

**基本的な構造**
```
<section>
  <!-- セクション固有のコンテンツ -->
</section>
```
この構造では、section タグを使用して文書内の意味的なセクションを作成します。開始タグ `<section>` と終了タグ `</section>` の間に、そのセクションに特有のコンテンツを配置します。

**セクションのタイトル**
`<section>` 要素は通常、タイトルを持つ。これは `<h1>`、`<h2>`、`<h3>` などの見出しタグを使用して表現されます。
```
<section>
  <h2>セクションのタイトル</h2>
  <!-- セクション固有のコンテンツ -->
</section>
```

**複数のセクションを使用する**
ウェブページには、通常、複数の section 要素が含まれる。それぞれの section は、異なるトピックやグループ化されたコンテンツを表します。
```
<section>
  <h2>最初のセクション</h2>
  <!-- コンテンツ -->
</section>

<section>
  <h2>次のセクション</h2>
  <!-- コンテンツ -->
</section>
```

**使用上の注意**
・sectionはセマンティックな要素であり、ページの意味的な構造を強化するために使うべき。
・すべてのコンテンツをsectionで囲む必要はない。セクションが独立して意味を持つ場合にのみ使用すること。
・レイアウトのためだけにsectionを使用するのは避け、そのような場合はdivなどの他の要素を使用するほうが適切。

---
**`<article>`タグの基本的な使い方**
article タグは、HTML5で導入されたセマンティック（意味的な）要素で、独立した、自己完結型のコンテンツを表します。これには、例えばニュース記事、ブログの投稿、フォーラムのスレッド、ユーザーコメントなどが含まれます。article 要素内のコンテンツは、文書の他の部分とは独立していても意味を持ちます。

**基本的な構造**
```
<article>
  <!-- 記事のコンテンツ -->
</article>
```
この構造では、article タグを使用して独立したコンテンツのブロックを作成します。開始タグ `<article>` と終了タグ `</article>` の間に、記事固有のコンテンツを配置します。

**使用上の注意**
・articleは、独立して配信または再利用が可能なコンテンツを囲むために使用される。
・各articleは、通常は独自のヘッダー、フッター、またはその他の識別情報を持っている。
・article 内に別の article をネストすることは可能だが、ネストされた article は親要素のコンテンツの一部として独立しているべき。例えば、ブログ投稿内のコメントなどがこれに該当する。

**`<article>`と`<section>`の違い**
`<article>`と`<section>`は似ているが、用途が異なる。article は独立して意味を持つコンテンツのために使用され、section は文書内の関連するコンテンツをまとめるために使用される。article は<u>単独で存在し得るコンテンツ （例：ニュース記事、ブログ投稿）</u>に最適で、section は<u>文書内の意味的なグループ化（例：書籍の章、ウェブサイトの異なるセクション）</u>に適している。

`<article>`の実際の使用例
```
<article>
  <header>
    <h2>ブログのタイトル</h2>
    <p>投稿日：2024年1月17日</p>
  </header>
  <p>ここにブログの本文が入ります。</p>
  <footer>
    <p>著者：John Doe</p>
  </footer>
</article>
```
この例では、記事のタイトル、投稿日、本文、著者情報が含まれており、それぞれが :`<header>`、:`<p>`、:`<footer>` などのタグで構成されている。これにより、記事が独立したコンテンツとして明確になる。
